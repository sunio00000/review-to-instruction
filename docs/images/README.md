# Demo Images

This directory contains demo screenshots and GIFs for the README.

## Required Images

### 1. `demo.gif`
Animated demonstration showing:
- Opening a GitHub PR page
- Scrolling to a convention-related comment
- Clicking the "Convert to AI Instruction" button
- Button state changes (Processing → Converted!)
- Clicking "View PR →" link

**Recommended size**: 800-1000px width
**Format**: GIF or MP4 (converted to GIF)
**Duration**: 10-15 seconds

### 2. `pr-comment.png`
Screenshot of a GitHub PR comment with:
- Convention-related content (example: naming convention)
- Code examples with ✅/❌ markers
- "Convert to AI Instruction" button visible

**Recommended size**: 600-800px width
**Format**: PNG
**Include**: GitHub UI context (author, timestamp)

### 3. `generated-pr.png`
Screenshot of the auto-generated PR showing:
- PR title (e.g., "Add AI instruction: Component Naming")
- File changes (e.g., `.claude/instructions/component-naming.md`)
- PR description with convention details
- Files changed tab

**Recommended size**: 600-800px width
**Format**: PNG
**Include**: GitHub PR UI context

## Tools for Creating Demo Materials

- **Screen Recording**: OBS Studio, ScreenToGif, LICEcap
- **GIF Optimization**: ezgif.com, gifsicle
- **Screenshot Annotation**: Snagit, Greenshot, ShareX

## Contribution

To contribute demo materials, please ensure:
- Images are clear and high-quality (but optimized for web)
- UI elements are visible and not cut off
- No sensitive information (tokens, private repos) is shown
- Dark mode screenshots are preferred for consistency
